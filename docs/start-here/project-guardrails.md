# Project Guardrails

## Purpose

These guardrails explain what the repository license covers, how attribution
works, and how to avoid implying rights over third-party material.

This page is project policy for contributors. It is not legal advice.

## License Model

The repository uses a dual-license model:

- Original documentation, diagrams, templates, walkthroughs, checklists, and
  other non-code educational content are licensed under CC BY 4.0.
- Original source code, lab implementations, scripts, tests, software examples,
  configuration, and automation are licensed under the MIT License.
- Third-party materials are not relicensed by this repository.

The repository `LICENSE` file is the authoritative license notice.

## Attribution Boundaries

The project license applies only to original work created for this repository.
It does not apply to:

- linked articles, documentation, talks, videos, books, or courses;
- quoted external text;
- external diagrams, screenshots, tables, or images;
- vendor names, trademarks, logos, or product screenshots;
- standards, specifications, or documentation owned by another organization;
- any third-party file that carries its own license notice.

Any included third-party file must carry its own source and license notice.

If a page links to or cites an external source, that source remains under its
own terms. A citation is not permission to copy, relicense, or closely
paraphrase the source.

## External Source Attribution

Cite external sources when a factual claim depends on them. Examples include:

- protocol behavior;
- vendor product limits;
- standards or specifications;
- incident reports or public case studies;
- legal, compliance, privacy, or security claims;
- benchmark numbers or measurements.

Keep citations specific. Link the source near the claim it supports, and avoid
using a citation as a replacement for original explanation.

## Original Examples And Diagrams

Examples and Mermaid diagrams should be original to this project. A contributor
may learn from external sources, but the final content should use new examples,
new wording, and new diagram structure.

If a reader could identify the source from the scenario, sequence, labels,
layout, or exercise structure, rewrite the material from scratch.

## When This Matters

These guardrails matter when:

- adding a docs page based on research;
- adapting a design pattern into a new example;
- creating Mermaid diagrams;
- building labs from common system behavior;
- quoting or linking external sources;
- adding code examples or scripts.

## Checklist

Before publishing:

- Confirm the material is original to this repository.
- Confirm third-party sources are cited only for specific factual claims.
- Confirm copied screenshots, tables, diagrams, exercises, and source structure
  are not included.
- Confirm code-oriented files and educational content are covered by the right
  repository license category.
- Confirm third-party works are not described as owned or relicensed by this
  project.
